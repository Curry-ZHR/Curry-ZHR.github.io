Git的相关分享

在我们学习编程语言的过程中，我认为有一个点我们无法绕开，它在我们编写代码完成后扮演着极其重要的作用。下面我来分享一下近期我所学习、了解到的有关Git的知识。（~~被迫营业~~）

概念

Git是一种分布式版本控制系统，相较于其他的集中式版本控制系统，它的优势主要体现在安全性（每个人都拥有自己的版本库）、强化责任（每一个改动部分必须提交改动日志，在两个人编代码时保持代码的一致性）等方面，而且它无需联网即可工作，没有中央服务器，结束了集中式版本控制系统提交文件必须联网并经历漫长等待的历史。此外，Git依赖于它强大的分支管理能力，使广大开发人员钟情于它。

分支管理

所谓分支管理，我们可以把它理解为每一个分支相当于一个平行宇宙，正如生活在平行宇宙中的人毫无交集，我们可以在每一个分支中分别编写程序，而互不影响。但有一点与平行宇宙不同的是，当我们把dev分支写好后我们可以将其合并到master分支上，以确保我们的master分支相对安全，我们肯定都不想遇上master分支崩溃时自己写好的大串代码因未保存而付诸东流。此外，分支管理的强大之处在于每个开发人员都可以在属于自己的分支中工作，利用并行开发节约大量的开发时间成本，分支相当于临时保存代码的地方，我们随时想提交到master分支均可，git能够整合上千人员所写的代码，所以我们无需担心dev分支无法提交。正是git的这一绝对优势，使得它在众多版本控制系统中脱颖而出。

分支策略

master分支十分稳定，仅用来发布我们写好的代码成果。通常我们干活都是在dev分支上，如果我们开发的项目庞大，是由多人协作完成的，那我们考虑在dev分支的下级再创建一个分支，我们在这个分支上进行工作，然后上传到dev分支进行初步合并然后检查代码是否可行，若无问题再考虑上传到master分支上。

git的相关命令行

下面是我收集到的git的相关命令行，有需要的同学可以记一下
$git checkout<文件名> 创建分支
$git switch -c dev 由当前分支切换到dev分支
$git branch 列出所有分支并查看
$git merge 合并指定分支到当前分支（merge后面是需要跟指定分支的名字吗？）
$git branch -d 删除这个分支
$git fetch 从另一版本库中下载文件
$git pull 将远程分支合并到本地分支
$git push 将本地分支合并到远程分支
$git log -graph 看当前的分支合并图
$git add+文件名 添加文件到本地仓库 然后上传到github
$git commit -m”conflict fixed” 手动解决冲突之后检查冲突是否还存在

git合并冲突的原因及如何解决问题

1 两个分支中修改了同一个文件
2 两个分支中修改了同一个文件的名称
解决冲突方法：手动解决，修改冲突的代码只能存在一个。（但具体怎么修改我还不是非常清楚）
